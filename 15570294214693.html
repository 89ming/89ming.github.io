在你 git 发布出问题的时候

2019/5/5

git小白的学习记录

通常在用终端发布脚本命令的时候出现红色黄色的warning时

再也不怕终端发布后出现问题

  1. 用终端进入生成网站的文件夹

    如果不知道怎么添加路径 直接来到网站文件夹的上一个文件夹拷贝文件夹（注意不是复制）就能拷贝到路径，粘贴到cd

  2. 删除 “我的blog” 文件夹，然后在命令行中执行：git clone '你的 git pages 的 库' '我的blog'。 这里的 '你的 git pages 的 repo' '我的blog' 请换成自己的。

    以防记不住库地址最好的办法就是复制库地址。删除“我的blog”这一步不用担心，mweb随时能够帮你再生成一个网站文件夹。执行git clone这一步是把你的库里文件复制到本地，github库里有没有文件都没关系，因为之后会通过mweb生成网站，就会把本地文件生成到这个文件夹

  3. 点击mweb网页文件夹右键里的清理并重新生成网站

    很重要的一步
